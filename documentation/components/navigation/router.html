A Router implementation for a View that is displayed as a route. Query Parameters are recognized and available in the View as Property "queryParams". When the Router is nested the "level" Attribute has to be set. Every router has its own hashtag with a string that points to the javascript file. Wenn der Router verschachtelt ist, so muss er das "level" Attribut besitzen. The router also includes another component, the routes in which is stored, to which File the corresponding URL belongs.
