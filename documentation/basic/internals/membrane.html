A membrane is a wrapper around an entire object graph, as opposed to a wrapper for just a single object. Typically, the creator of a membrane starts out wrapping just a single object in a membrane. The key idea is that any object reference that crosses the membrane is itself transitively wrapped in the same membrane. Object references typically cross the membrane in one direction (out->in) by passing parameters to an object’s method, and in the opposite direction (in->out) by being returned as a value from a method or by being thrown as an exception. Eine Membran ist ein Wrapper um einen ganzen Objektgraphen, im Gegensatz zu einem Wrapper für nur ein einzelnes Objekt. Typischerweise beginnt der Ersteller einer Membran damit, nur ein einzelnes Objekt in eine Membran einzuwickeln. Die Schlüsselidee ist, dass jede Objektreferenz, die die Membran durchquert, selbst transitiv in dieselbe Membran eingewickelt wird. Objektreferenzen durchqueren die Membran typischerweise in einer Richtung (out->in), indem sie Parameter an die Methode eines Objekts übergeben, und in die entgegengesetzte Richtung (in->out), indem es als Wert von einer Methode zurückgegeben oder als Ausnahme ausgelöst wird. To ease speaking about membranes, I will use the term “wet objects” for objects that live inside of the membrane, and the term “dry objects” to refer to objects that live outside of the membrane. The key thing about membranes is that they convert any outbound reference to a wet object that crosses them into a proxy, which is a dry object representing the corresponding wet object on the other side of the membrane. Similarly, the membrane converts any inbound dry object reference into a proxy which is a wet object that represents the corresponding dry object on the other side of the membrane. Um das Sprechen über Membranen zu erleichtern, werde ich den Begriff „wet Objects“ für Objekte verwenden, die innerhalb der Membran leben, und der Begriff „dry Objects“ bezieht sich auf Objekte, die außerhalb der Membran leben. Das Wichtigste an Membranen besteht darin, dass sie jeden ausgehenden Verweis auf ein wet Object, das sie kreuzt, in einen Proxy umwandeln, der ein dry Object ist die das entsprechende wet Object auf der anderen Seite der Membran darstellt. Ebenso wandelt die Membran jede um eingehende dry-objektreferenz in einen Proxy, der ein wet Object ist, das das entsprechende dry Object auf dem darstellt andere Seite der Membran.
