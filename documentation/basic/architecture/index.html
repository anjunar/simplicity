Architecture Architektur The architecture is designed to use native components as much as possible, that's why this framework can also be seen as an extension of the HTML standard. For the web components note that the Safari browser does not fully implement Web Components, so the framework works only partially in the Safari browser. Die Architektur ist so ausgelegt, dass Sie so viel auf native Komponenten zurückgreift wie möglich, deswegen kann dieses Framework auch als eine Erweiterung des HTML Standards gesehen werden. Bei den Web Components ist zu beachten, dass der Safari Browser nicht komplett Web Components implementiert hat, deswegen funktioniert das Framework nur teilweise im Safari Browser. The Web Components are loaded by the ES6 Modules, which corresponds to a Javascript file. Thereby the ES6 Modules are used, there is no build management that compile all javascript files in one single file. This has the advantage that all modules are loaded asynchronously lazy, if they are needed. Die Web Components werden durch die ES6 Modules geladen, was einer Javascript Datei entspricht. Dadurch das ES6 Modules benutzt werden gibt es kein Build management das am Ende alle Javascript Dateien zu einer einzigen Datei kompiliert. Das hat den Vorteil das alle Module asynchron lazy geladen werden, wenn Sie gebraucht werden. In contrast to the common frameworks such as Angular, React and Vue, this framework does not use any Dirty checking mechanism to keep components up to date. Instead, events are set on the model. If the corresponding element leaves the DOM, it automatically de-registers the event handler. Everything works via OnPush, since there is no dirty checking so that the entire component graphs must be traversed. Im Gegensatz zu den gängigen Frameworks wie z.B. Angular, React und Vue benutzt dieses Framework kein Dirty Checking, um die Komponenten aktuell zu halten. Stattdessen wird auf Events gesetzt die am Model registriert werden. Verläßt das entsprechende Element den DOM, wird automatisch der Event-handler de-registriert. Somit funktioniert alles via OnPush, da es kein Dirty Checking gibt, dass den gesamten Komponenten Graphen durchlaufen muss. Because each model is surrounded by a membrane, tracking changes is easy. Changes to the model will automatically fire all appropriate handlers that updates the components. Dadurch das jedes Model durch eine Membrane umgeben wird, ist das tracken von Changes einfach. Ändert sich das Model werden automatisch alle entsprechenden Handler gefeuert, dass zum Aktualisieren der Komponenten führt.
